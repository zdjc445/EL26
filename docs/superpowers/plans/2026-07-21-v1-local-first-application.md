# Time V1 本地优先主体应用实施总计划

状态：已批准

## 目标

实现产品规格中的 `AUTH-01`～`AUTH-03`、`UI-01`～`UI-03`、`KB-01`～
`KB-06`、`RAG-01`～`RAG-06`、`CHAT-01`～`CHAT-03`、`CAL-01`～
`CAL-08` 和 `LEARN-01`～`LEARN-04`。领域与应用服务采用生产目标语义；当前
通过 ADR-0004 的 `memory` profile 提供完整本地运行能力。

本计划不声称完成真实供应商、生产数据库、生产 RAG 质量、容量、备份或发布
门禁。它们在准确契约和资源存在后通过独立计划实现。

## 架构规则

1. 后端继续使用顶层包 `time_agent`。
2. 每个领域模块包含 `domain`、`application`、`ports`、`adapters` 和 `api` 边界；
   领域层不得导入 FastAPI、Pydantic、LangChain、LangGraph 或具体适配器。
3. 跨模块调用只通过公开应用接口和不透明 ID，不导入其他模块内部实体。
4. `time_agent.bootstrap.runtime` 是唯一适配器组合根。
5. 所有仓储方法显式接收权威 `user_id`，HTTP 不接受客户端提供的权威用户 ID。
6. 所有时间、随机数、ID、模型输出和外部失败在测试中可控。
7. 公开 API 固定在 `/api/v1`，每个任务同步 OpenAPI 与生成 TypeScript 类型。
8. 每个功能先写失败测试，确认失败原因，再实现最小行为并运行完整相关门禁。
9. 内存开发辅助接口必须同时满足 `TIME_RUNTIME_PROFILE=memory` 与
   `TIME_ENVIRONMENT=local`；其他配置返回 404。
10. 未获得真实契约前不创建 `production` profile 或供应商专有字段。

## 交付序列

### Milestone 1：共享运行时、身份与工作台

需求：`AUTH-01`～`AUTH-03`、`UI-01`～`UI-03`。

产出：

- `MemoryRuntime`、时钟、ID、Unit of Work 和配置选择；
- 邮箱/手机号标识、验证码挑战、密码哈希、会话、撤销和认证中间件；
- 内存验证码 Outbox 的开发读取接口；
- 登录、注册、找回密码、设置页和双侧栏工作台；
- 刷新后保持的独立折叠状态、窄屏互斥抽屉和可访问性语义；
- 认证暴力尝试、会话固定、撤销复用和日志脱敏测试。

详细子计划在实现前固定准确模型字段、API Schema、依赖版本、文件路径、RED
测试和提交序列。

### Milestone 2：知识空间、上传与解析

需求：`KB-01`～`KB-06`。

产出：

- 逻辑文件夹树、循环约束、移动历史和收件箱；
- 200 MB 可配置限制、分片上传会话、校验值和不可覆盖资料版本；
- PDF 文本层与 DOCX 标题、段落、表格解析；
- 资料版本状态机、有限重试、失败原因和部分可用范围；
- 整理提案的编辑、批准、拒绝与原子目录更新；
- 恶意文件名、扫描 PDF、空文本、越权和删除残留测试。

### Milestone 3：本地混合检索与引用

需求：`RAG-01`～`RAG-06`。

产出：

- 确定性内容分块和版本化索引；
- 中文/英文词项稀疏召回与确定性哈希稠密向量；
- 用户与文件夹/资料/章节强制过滤；
- RRF 融合、确定性重排、来源优先级、上下文预算和低证据判定；
- 行内引用、冲突标记和完整检索追踪；
- 两用户相似资料、提示词注入、缺失答案和范围逃逸测试。

本地检索只证明业务管线和隔离，不替代 ADR-0003 的生产 Recall@10 与引用质量
评测。

### Milestone 4：对话、学习会话与派生知识

需求：`CHAT-01`～`CHAT-03`、`LEARN-03`、`LEARN-04`。

产出：

- 原始会话与消息、SSE 事件序列和断线重连游标；
- 确定性 Agent 状态图：目标、已有理解、分段讲解、理解检查、调整和总结；
- 掌握证据、用户纠正与复习时间；
- 会话整理提案及逐项编辑、合并、批准、拒绝；
- 派生条目来源消息、资料引用、审批和修改历史；
- 删除派生条目不删除原始对话的测试。

不得把确定性模型适配器的结果声明为真实 LLM 质量。

### Milestone 5：内部日历与提醒

需求：`CAL-01`～`CAL-08`。

产出：

- 定时/全天、普通/学习、单次/重复事件；
- 月、周、日、列表查询以及关键词、类型、状态过滤；
- 系列规则、例外、跳过、本次、本次及后续、全部变更；
- 冲突、空闲时间、多提醒、站内/浏览器投递与有限重试；
- 幂等键、期望版本、来源消息、前后值审计和补偿撤销；
- Agent 读取、明确单事件直写、歧义澄清和高影响审批状态机；
- 时区、夏令时、重复范围、并发、乱序和越权测试。

### Milestone 6：学习任务与原子排期

需求：`LEARN-01`、`LEARN-02`。

产出：

- 学习任务目标、原因、预计用时、进度、截止时间和来源资料；
- 一个任务拆分多个学习时段；
- 基于空闲时间、冲突和截止时间的确定性排期提案；
- 编辑、批准、拒绝和原子批量日历提交；
- 失败不留下半套计划、重复批准幂等和补偿撤销测试。

### Milestone 7：完整工作台与本地 V1 闭环

需求：全部 V1 需求。

产出：

- 今天、对话、日历、知识空间、学习进度和设置页面；
- PDF/DOCX 上传到学习、派生内容和排期的浏览器闭环；
- 每个 API 的跨用户回归矩阵；
- 需求编号到测试文件的可执行追踪报告；
- OpenAPI、容器、文档、E2E、安全和许可证完整验证。

## 每个 Milestone 的强制流程

1. 写准确的详细子计划并标记为 `草案`；
2. 用户书面批准后改为 `已批准`；
3. 创建 `phase/<phase-id>-<scope>` 隔离 worktree；
4. 每个任务执行 RED → GREEN → 重构 → 相关验证；
5. 每个任务单一目的提交；
6. 完成自审、独立 Review 或用户明确的单次豁免；
7. `python tools/project.py verify`、容器构建和 CI 全绿；
8. 按用户当前授权直接 fast-forward 到 `main`，然后恢复并核验分支保护。

## 全局验证

- 单元：领域状态、权限、幂等、状态机和排序算法；
- 属性：时间、重复规则、文件夹、分块、乱序和并发；
- 集成：FastAPI、OpenAPI、内存事务、SSE 和完整应用组合；
- E2E：产品规格第 12 节的十步闭环；
- 安全：每个仓储、对象、索引和 HTTP 入口的跨用户访问为 0；
- 可访问性：键盘、焦点、名称、状态和窄屏抽屉；
- 漂移：OpenAPI 与 TypeScript 类型可重复生成；
- 供应链：锁文件、许可证、依赖和镜像扫描。

## 明确延后但保留切换边界

- 邮件与短信真实供应商；
- PostgreSQL、S3、RabbitMQ、Valkey、Qdrant 与 Celery 的真实适配器；
- 阿里云百炼 LLM、Embedding、Rerank 和 LangGraph 持久化 Checkpoint；
- 生产 RAG 固定评测、容量、故障注入、备份恢复和 Terraform；
- 生产发布、密钥、真实用户数据和生产监控。

这些项目未完成前只能声明“本地 V1 主体应用完成”，不能声明 V1 可生产发布。

## 总完成条件

- 七个 Milestone 的详细子计划均获批准并实施；
- 产品规格第 12 节可在 `memory` profile 下由浏览器完成；
- 全部需求编号存在可执行测试映射；
- 跨用户访问、未经批准的高影响写入和伪造资料引用均为 0；
- 完整仓库验证、两个容器构建和 `main` CI 全绿；
- 工作区干净，文档明确区分本地主体完成与生产门禁未完成。

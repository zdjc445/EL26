# ADR-0004：运行配置与内存适配器边界

状态：已批准

日期：2026-07-21

## 背景

V1 的业务主体需要在邮件、短信、PostgreSQL、对象存储、RabbitMQ、Valkey、
Qdrant 和模型供应商尚未接入时继续实现和验证。用户已明确要求全部 V1 需求
先使用内存基础设施运行，同时保留以后切换真实基础设施的单一配置入口。

内存实现不能成为一套与生产实现不同的业务逻辑，也不能用空 Mock 伪造成功。
供应商字段、错误码和资源标识在真实契约不存在时不得猜测。

## 决策

采用端口与适配器边界：

- 领域模型和应用服务只依赖 `typing.Protocol` 定义的端口；
- HTTP、Agent、Worker 和调度入口只调用应用服务，不直接访问适配器；
- 内存适配器实现完整的查询、约束、幂等、并发版本、事务模拟、失败状态和
  用户隔离语义；
- 所有内存适配器由同一个进程级 `MemoryRuntime` 持有，测试可以显式创建隔离
  runtime；
- `TIME_RUNTIME_PROFILE` 是唯一运行配置入口，当前只接受精确值 `memory`；
- 未注册的 profile 在应用启动时立即失败，禁止静默回退到内存实现；
- 未来只有在真实适配器、迁移、契约测试和运行手册全部存在后，才新增
  `production` profile；
- 业务 ID 使用不透明 UUID，时间和随机数通过显式端口注入；
- API Schema、领域命令和审计事件不包含供应商专有字段；
- 文件内容、检索向量、队列任务和会话状态可以保存在内存适配器中，但业务
  状态机与生产目标保持一致。

## 一键切换的含义

“一键切换”只表示组合根按一个配置值选择一组已经注册且验证过的适配器：

```text
TIME_RUNTIME_PROFILE=memory
→ MemoryRuntime
→ 内存仓储、对象存储、任务队列、检索索引、通知和模型适配器
```

未来真实基础设施完成后：

```text
TIME_RUNTIME_PROFILE=production
→ ProductionRuntime
→ PostgreSQL、S3、RabbitMQ、Valkey、Qdrant 和已批准供应商适配器
```

在 `ProductionRuntime` 尚不存在时，配置 `production` 必须报错。不得提交返回
固定成功值的占位生产适配器。

## 内存事务语义

跨实体写入通过 `UnitOfWork` 端口执行。内存实现持锁复制待修改状态，全部校验
通过后一次替换；异常时丢弃副本。它必须验证：

- 用户范围；
- 唯一约束；
- 期望版本；
- 幂等键；
- Outbox 与业务写入的原子性；
- 批量日历写入全有或全无。

内存实现不声称具备跨进程持久化、崩溃恢复或生产容量。相关能力只能由真实
基础设施集成测试证明。

## 外部能力的本地实现

- 邮件和短信：验证码投递到用户作用域的内存 Outbox；测试和开发 API 只能在
  `memory` profile 下读取，响应和日志不直接包含验证码；
- 对象存储：实现分片、完成、校验值、不可覆盖版本和删除语义；
- 任务队列：实现入队、领取、确认、失败、有限重试和死信状态；
- 检索：实现用户强制过滤、稀疏召回、确定性哈希稠密向量、RRF、重排、范围
  限制和追踪；
- 模型：实现可编程的确定性学习模型适配器，用于真实状态图和审批流程，不将
  其质量结果冒充供应商模型评测；
- 通知：实现站内和浏览器通知投递状态，不实现邮件或短信日程提醒。

## 后果

正面：

- 全部业务行为可在无外部账号时运行和 E2E 验证；
- 真实适配器不会侵入领域与 HTTP 契约；
- 多租户、幂等、审批和恢复规则可以先得到确定性测试；
- 供应商契约缺失不会诱导代码猜测字段。

负面：

- 进程重启会丢失内存数据；
- 内存事务不能证明 PostgreSQL 并发和迁移行为；
- 本地检索与模型不能证明生产 RAG 质量门槛；
- 真实基础设施仍需独立 ADR、实施计划和契约测试。

## 替代条件

当某项真实基础设施具备已批准 ADR、固定契约、可重复集成测试、迁移和回滚
证据时，为该端口新增真实适配器。不得删除内存适配器；它继续用于单元、组件
和本地 E2E。

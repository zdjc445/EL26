# Time 首版产品规格：学习驱动型个人 Agent

状态：已批准

版本：1.0

日期：2026-07-17

## 1. 产品定位

Time 是面向约 1000 名大学生的 Web 学习与行动 Agent。知识资料提供学习依据，Agent 负责讲解、检查理解、整理学习产物，并将学习任务与普通事务统一安排到内部日历。

核心闭环：

```text
上传资料
→ 自动提出整理建议
→ 用户批准归档
→ 基于资料讲解
→ 检查理解
→ 提炼笔记、疑问、错题和任务
→ 用户批准派生内容
→ 学习任务安排到日历
→ 根据后续学习结果继续调整
```

日程和学习不是两个互不相关的工具。首版的产品主轴是学习，普通日程作为同一日历中的基础能力存在。

## 2. 用户与规模

- 目标用户：大学生；
- 首版规模：约 1000 个注册用户；
- 资料容量假设：每名用户数 GB，总原始资料为 TB 级；
- 客户端：桌面 Web 优先，同时支持窄屏自适应；
- 后端核心语言：Python；
- Web 前端：TypeScript；
- 部署：Vultr 日本地域；
- 不以中国大陆网络可达性作为首版约束。

用户存储额度是运营配置的账号权益，不在首版写死统一数值。所有上传在创建上传会话前检查当前账号权益。

## 3. 账号与认证

### AUTH-01 注册

用户必须使用邮箱或手机号完成验证码验证后创建账号。账号至少保留一个已验证的登录标识。

### AUTH-02 登录

已验证的邮箱或手机号均可使用验证码登录。用户设置密码后，也可以使用密码登录。

### AUTH-03 凭证管理

- 首次注册不能仅凭密码，必须先验证邮箱或手机号；
- 添加或变更登录标识时重新验证；
- 邮箱和手机号全局唯一；
- 支持设置、修改和找回密码；
- 登录会话可由用户主动退出，也可由服务端撤销；
- 验证码具有有效期、发送频率限制和尝试次数限制；
- 密码只保存安全哈希，日志不得记录密码、验证码或完整会话凭证。

## 4. Web 信息架构

### UI-01 全局导航

首版采用“学习工作台”布局，主导航包含：

- 今天；
- 对话；
- 日历；
- 知识空间；
- 学习进度；
- 设置。

### UI-02 学习工作台

工作台同时展示：

- 全局导航；
- 当前资料和历史对话上下文；
- Agent 对话；
- 当前学习目标、来源、待整理内容和建议任务。

### UI-03 双侧栏折叠

- 全局导航可折叠为图标栏，不能完全消失；
- 资料与对话上下文栏可以完全收起，内容区边缘保留展开按钮；
- 两种折叠状态独立保存，刷新和切换页面后保持；
- 窄屏下改为覆盖式抽屉，打开一层时关闭另一层；
- 所有控制支持键盘和屏幕阅读器，并暴露明确的展开／收起状态。

## 5. 内部日历

首版日历是产品内部的唯一日程事实来源，不接入外部日历，不导入课表。数据模型保留来源和外部标识扩展边界，但首版不实现同步。

### CAL-01 事件类型

- 定时事件；
- 全天事件；
- 单次事件；
- 重复事件；
- 普通事件；
- 与学习任务关联的学习时段。

### CAL-02 基础操作

用户可以创建、查看、修改、取消、完成和删除事件，并按日期、关键词、类型和状态查询。

### CAL-03 事件内容

事件至少表达标题、说明、开始时间、结束时间、时区、全天状态、地点或链接、状态、来源和审计信息。

### CAL-04 重复规则

重复事件使用“事件系列 + 例外”模型，支持：

- 重复到指定日期或次数；
- 跳过某一次；
- 只修改或取消某一次；
- 修改本次及后续；
- 修改整个系列。

系统不能通过无约束复制大量普通事件模拟重复规则。

### CAL-05 提醒

首版支持站内提醒和浏览器通知。一个事件可以有多个提醒。系统记录计划时间、投递状态、失败原因和重试结果。

### CAL-06 视图

提供月、周、日和列表视图，支持拖拽调整时间、调整持续时间、筛选学习／普通事件和显示冲突。

### CAL-07 冲突与空闲时间

系统可以检测事件重叠、查询空闲时间，并在学习排期时避开已有事件。

### CAL-08 Agent 操作策略

- 读取和查询直接执行；
- 信息完整、无歧义的单个事件创建可直接执行，随后显示结果和撤销入口；
- 时间、对象或含义不明确时先向用户澄清；
- 修改、删除、重复系列变更和批量操作必须展示影响范围及前后差异，并等待用户明确批准；
- 真正写入由确定性应用服务完成，模型只能生成操作草案；
- 每次写操作必须幂等、进行并发版本检查并记录来源消息及前后值；
- 撤销通过补偿操作实现，审计历史不能被删除。

## 6. 学习任务与日历关系

### LEARN-01 概念分离

学习任务描述“学什么、为什么、预计用时、进度和来源资料”；日历事件描述“什么时候学习”。一个学习任务可以拆分为多个学习时段，普通事件可以不关联学习任务。

### LEARN-02 排期提案

Agent 可以结合截止时间、预计用时、空闲时间和现有冲突生成排期提案。批量日历写入前必须由用户批准，并以原子方式提交。

### LEARN-03 引导式讲解

Agent 的主要职责是讲解资料内容。一次学习会话包括：

1. 明确当前资料、主题和学习目标；
2. 获取学生已有理解；
3. 分段讲解核心概念、例子、定义或推导；
4. 穿插理解检查；
5. 根据回答调整讲解深度；
6. 总结知识、薄弱点和后续行动。

### LEARN-04 掌握证据

系统记录已接触概念、回答、错题、错误认识和复习时间作为掌握证据，不能仅因用户点击“懂了”就标记完全掌握。用户可以查看和纠正学习状态。

## 7. 知识空间与资料

### KB-01 逻辑文件夹

用户看到的是数据库中的逻辑目录树，不是服务器物理目录。移动和重命名只修改元数据，不移动底层对象。

### KB-02 首版文件类型

首版只接受：

- 具有可提取文本层的 PDF；
- DOCX。

不开放图片上传，不实现 OCR，不处理扫描 PDF 中的图像文字、手写、图片公式或图片表格。解析器保留扩展接口，后续增加 OCR 时不得改变现有资料和索引的身份模型。

### KB-03 上传限制

- 单文件最大 200 MB，限制作为运行配置而非业务代码常量；
- 浏览器通过预签名 URL 分片直传对象存储；
- API 负责创建上传会话、校验完成状态和投递处理任务；
- 原始对象不可被新上传或重新解析覆盖；
- 相同资料的重新上传和新版本必须有明确关系。

### KB-04 文本解析

- PDF 提取文本层、页码和可确定的来源位置；
- DOCX 提取标题层级、段落和可读取的表格文本；
- 无文本、文本层异常或仅包含扫描图像的文件标记为“不支持的扫描资料”；
- 失败文件不能生成空索引并显示为成功；
- 原始文件、解析结果、内容块和索引分别版本化。

### KB-05 自动整理

上传完成后，文件先进入收件箱。Agent 生成整理提案，包括建议目录、显示名称、标签、课程归属和重复关系。用户可以编辑、批准或拒绝。批准前不能静默移动文件。

### KB-06 资料归属

一份资料当前归属一个逻辑文件夹并保留移动历史。派生学习条目可以关联多个文件夹。

## 8. 对话与派生知识

### CHAT-01 原始对话

原始对话独立保存，不因知识整理而搬入文件夹或被改写。

### CHAT-02 会话整理提案

学习会话结束后，Agent 可以生成：

- 会话总结；
- 知识笔记；
- 未解决问题；
- 错题；
- 错误认识；
- 学习任务。

用户可以逐项编辑、选择目标文件夹、合并重复内容、批准或拒绝。未批准内容不进入默认 RAG。

### CHAT-03 可追溯性

每个派生条目记录来源会话、具体消息、引用资料、创建方式、审批状态和修改历史。删除派生条目不能删除原始对话。

## 9. RAG 与回答规则

### RAG-01 检索范围

用户可以限定某个文件夹、资料、章节或跨资料主题。所有检索由后端强制注入用户及范围过滤，不能依赖提示词隔离。

### RAG-02 混合检索

检索流水线包含查询处理、关键词召回、向量召回、元数据过滤、结果融合、重排和上下文组装。

### RAG-03 知识来源

来源优先级为：

1. 用户当前指定资料；
2. 用户批准的知识笔记；
3. 当前文件夹其他资料；
4. 模型通用知识补充。

### RAG-04 回答展示

回答采用统一连贯文本，不固定拆成“资料依据”和“补充讲解”两个区块。来自资料的结论使用行内引用标明文件和页码；模型补充不得伪造引用。资料之间存在冲突时必须明确指出。

### RAG-05 低证据处理

资料不足时允许模型使用通用知识补充，但不能声称补充内容来自用户资料。无法可靠回答时必须说明限制。

### RAG-06 回答追踪

后端记录每段回答使用的检索结果、资料版本、内容块、模型、提示词版本和工具调用，用于审计、调试和评测。

## 10. 外部 AI 服务

- 首版大模型、Embedding 和 Rerank 优先采用中国大陆供应商；
- 主候选为阿里云百炼／通义千问；
- 文本向量候选为 `text-embedding-v4`；
- 文本重排候选为 `qwen3-rerank`；
- 大模型使用经真实评测后固定的可用快照版本，不能在生产使用不受控的浮动别名；
- 模型、Embedding 和重排均经内部适配层调用；
- 文本从 Vultr 日本地域发送到中国大陆处理的跨地域数据流已被产品设计接受；
- 发送数据必须最小化，大模型只接收当前问题所需的资料片段。

## 11. 首版非目标

- OCR、图片上传、扫描 PDF 识别、手写识别、图片公式识别；
- 外部日历同步；
- 课表导入；
- 多人共享日历、会议邀请和资源预订；
- 邮件或短信日程提醒；
- 自动生成完整课程体系；
- 自动归档大量闪卡或练习题；
- 原生移动端或桌面客户端；
- 将第三方托管知识库作为唯一数据源；
- 首版微服务化或 Kubernetes 化。

## 12. 首版验收摘要

首版只有在以下完整闭环可由真实用户完成时才满足产品验收：

1. 使用邮箱或手机号验证码注册并登录；
2. 上传可解析的 PDF 或 DOCX，查看处理进度；
3. 审阅并批准文件整理提案；
4. 在指定资料范围内与 Agent 学习并获得可追溯引用；
5. 完成理解检查；
6. 审阅并批准会话总结、知识笔记、疑问、错误认识和学习任务；
7. 将学习任务安排到日历；
8. 通过对话和日历界面查询、修改和完成日程；
9. 修改、删除和批量操作遵守确认策略；
10. 全流程不存在跨用户数据访问。

详细质量、容量、恢复和发布门槛见工程文档。
